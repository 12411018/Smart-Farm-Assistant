## 🚀 HOW TO RUN (QUICK START)

### **Step 1: Prepare Environment**
```bash
# Install Python packages (if not done)
cd backend
pip install -r requirements.txt

# Install npm packages (if not done)
cd ..
npm install
```

### **Step 2: Initialize Database**
```bash
cd backend
python init_db.py
```

### **Step 3: Start All Services** 

Open 4 terminals:

**Terminal 1: Ollama (Local LLM)**
```bash
ollama serve
```

**Terminal 2: FastAPI Backend**
```bash
cd backend
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

**Terminal 3: React Frontend**
```bash
npm run dev
```

**Terminal 4: Monitor (optional)**
```bash
# View logs or test APIs
curl http://127.0.0.1:8000/health
```

### **Step 4: Access Application**
- **Frontend**: http://localhost:5173/
- **API Docs**: http://127.0.0.1:8000/docs
- **Health Check**: http://127.0.0.1:8000/health

## 🔗 API ENDPOINT REFERENCE

### **Crop Planning**
- `POST /crop-plan/create` - Create new plan
- `GET /crop-plan/{id}` - Get plan details
- `GET /crop-plan/user/{userId}` - List user plans
- `DELETE /crop-plan/{id}` - Delete plan

### **Irrigation**
- `GET /irrigation/schedule/{planId}` - Get schedule
- `POST /irrigation/adjust` - Adjust based on weather
- `GET /irrigation/logs/{planId}` - Get logs

### **Intelligence**
- `POST /chat` - Chat with AI
- `GET /crop-insight/{planId}` - Get crop advice
- `POST /weather-analysis` - Weather + AI
- `GET /calendar/{planId}` - Calendar events

### **Sensors** (IoT Optional)
- `POST /sensor/raindrop` - Log rainfall
- `POST /sensor/dht11` - Log temp/humidity
- `POST /sensor/soil-moisture` - Log soil moisture

### **Health & Logs**
- `GET /health` - Health check
- `GET /all-logs` - Operation logs

## 🎓 LEARNING & EXTENSION

### **To Extend the System**:

1. **Add More Crops**
   - Edit `backend/crop_engine/crop_data.py`
   - Add crop to `CROPS` list in form

2. **Add New API Endpoints**
   - Create function in `backend/main.py`
   - Use `@app.get()` or `@app.post()` decorators
   - Return JSON response

3. **Add New Database Tables**
   - Create model class in `backend/models.py`
   - Add relationship to existing tables
   - Run `python init_db.py` again

4. **Add New Pages**
   - Create `src/pages/NewPage.jsx`
   - Create `src/styles/NewPage.css`
   - Import and add route in `src/App.jsx`
   - Update `Navigation.jsx`

5. **Customize Styling**
   - Edit `src/styles/globals.css` for colors/fonts
   - Edit component `.css` files for layout

## 🚨 TROUBLESHOOTING QUICK FIXES

| Problem | Solution |
|---------|----------|
| "Cannot connect to database" | Run `python init_db.py`, check PostgreSQL |
| "Backend not responding" | Restart backend: `uvicorn main:app --reload` |
| "Ollama model not found" | Run `ollama pull mistral:latest` |
| "Frontend shows errors" | Check browser console (F12), verify `.env` |
| "Crop plan won't save" | Check backend logs for database errors |
| "Chatbot not responding" | Ensure Ollama running, check `http://localhost:11434/api/tags` |
| "Firebase errors" | Firebase is optional, system works without it |
